Introduction

Lab 9 is a translation of Lab 6. Your filemaker.ps1 script is to create a file containing a specified number of records based on a record layout. The record layout consists of constants and verbs that when read and executed by your PowerShell script, will cause a file to be created that matches that format.

Most of the details of this lab are covered in Lab 6, so keep the Lab 6 writeup handy as you work through this assignment. This writeup provides the details specific to PowerShell and a summary of the details from Lab 6. This lab might be most quickly completed by copying over your Lab 6 Python code and translating it into PowerShell.

Requirements

Your PowerShell script shall be named filemaker.ps1, found in the root folder of your private rep and be marked executable.

Usage: ./filemaker.ps1 INPUTCOMMANDFILE OUTPUTFILE RECORDCOUNT

The INPUTCOMMANDFILE contains one command per line in the file. Your filemaker script shall support the following commands in the INPUTCOMMANDFILE:

HEADER "string"

writes "string" once at the top of the file
STRING "string" or STRING 'string'

writes the constant "string" to each record in the output. Strings can be enclosed in single or double quote marks.
WORD label "filename"

write a random word from file "filename"
INTEGER label min max

write a random integer between min and max inclusive
REFER label

write the item referred to by LABEL (see notes below)

The label identifier you see above is a string that is unique in this file from all other labels and is a way to reference something created earlier. For example, consider the following sample command file we will call lab9cmds:

COMMAND EXPLANATION

STRING '"'

Output a double quote (notice that it is a single quote followed by a double quote followed by a single quote)

WORD lastname "surnames.txt"

Output a random word from file "surnames.txt", labeled lastname

STRING ", "

Output a comma followed by a space

WORD firstname "firstnames.txt"

Output a random word from file "firstnames.txt", labeled firstname

STRING '","'

Output a double quote, comma and a double qoute

REFER firstname

Output the same random word in this record as the command labeled firstname

STRING "."

Output a period

REFER lastname

Output the same random word in this record as the command labeled lastname

STRING '@mail.weber.edu"\n'

Output @mail.weber.edu followed by a double quote and a newline

  • Check for errors

    • Issue an Usage message and exit(1) if the user did not specify all three parameters

    • Convert RECORDCOUNT from a string to an integer (call it $recordCount) and exit(1) if not numeric or not a positive integer.

    • Issue an appropriate Error message and exit(1) if an error occurs opening any of the files.

  • Preprocess headers and files: Do this only once. For each line in the commandfile:

    • HEADER: write the header string to the output file

    • For WORD commands, "slurp" the entire file as a single list into a dictionary called $randomFiles, using Get-Content. Use the name of the file as the key. exit (1) if the file is not readable.

  • Do the following steps for each record to be created:

    • Initialize in dictionary $randomData

    • Generate random data and populate dictionary $randomData using random INTEGER generators and file data in dictionary randomFiles

    • Generate output from commands and dictionary $randomData

PowerShell Hints

PowerShell has no direct equivalent to shlex, so we will use regular expressions to match the command name and extract the parameters of the command. As in Lab 6, do not attempt to extract the parameters with split(" "): quoted strings can contain imbedded spaces.

Verify the commandline

The built-in PowerShell variable $args is an array containing the commandline arguments typed by the user. If there are not 3 commandline parameters, issue a "Usage:" message and exit 1.

if ($args.length -ne 3) {
    write-host ("Usage: ./filemaker.ps1 <commandfile> <outputfile> <recordcount>")
    exit 1
}
Reading a file

Use Get-Content to read the input file and "slurp" the entire file into an array. Put it inside a try/catch. The -erroraction stop operand permits the catch block to execute if the open fails.

try {
    $inputCommands = Get-Content -path $args[0] -erroraction stop (1)
}
catch {
    write-host ("Error opening or reading command file: $($_)") (2)
    exit 1
}
1 Read the entire file named by $args[0] into array $inputCommands
2 Output an error message if 'Get-Content' fails. $_ is the system-generated error message, and $($_) inserts $_ into the string for display.
Creating a new file

Use New-Item to open your output file

try {
    $outputFile = $args[1]
    New-Item -path $outputFile -erroraction stop | out-null (1)
}
catch {
    write-host ("Error opening output file: $($_)") (2)
    exit 1
}
1 piping the output of New-Item to out-null gets rid of unnecessary messages from appearing on the terminal.
2 Output an error message if New-Item fails, with the system-generated output message.
Writing to an output file

  • Use Add-Content to write a string to a file

  • Translate the \t and \n characters to their PowerShell equivalents `t (backtick t) and `n (backtick n)

  • Prevent a newline from being automatically appended to the string being written

  • In the example below, I created a function called writeToFile() to perform the translations so I wouldn’t have to do it everywhere in my script I needed to output a string.

function writeToFile ($outputFile, $outputString) { (1)
    $outputString = $outputString -replace [regex]::escape("\t"), "`t" (2)
    $outputString = $outputString -replace [regex]::escape("\n"), "`n" (3)
    try {
        add-content -path $outputFile -value $outputString -nonewline (4)
    }
    catch {
        write-host "Write failed to file $($outputFile): $_"
        exit 1
    }
}
1 $outputFile is the name of the outputfile as provided on the commandline. $outputString is the string to be written.
2 Translating tabs to the PowerShell equivalent
3 Translating newlines to the PowerShell equivalent
4 Preventing an automatic newline with the -nonewline option
Using Random Numbers

  • Use Get-Random to obtain a random item from an array variable or to generate a random integer.

$randomWord = Get-Random -inputobject $arrayVariable { (1)
$randomInteger = Get-Random -min $integerMin -max $integerMax (2)
}
1 Store in $randomWord a randomly retrieved entry from array $arrayVariable.
2 Store in $randomInteger a randomly generated integer between $integerMin and $integerMax.
In the examples below, $command is a line read from the commandfile. In the regular expressions, \w matches alphanumeric characters, "\s" matches whitespace, "^" matches the beginning of the string and "$" at the end matches the end of the string.
Matching HEADER

  • Match the HEADER command followed by a string in "double quotes".

if ($command -match '^HEADER\s+"(.*)"$') { (1)
	writeToFile $outputFile $matches.1 (2)
}
1 Search the $command variable for “HEADER” followed by whitespace followed by something in double-quote marks.
2 Write the string inside the double quote marks out to the output file by calling my convenient function I wrote called writeToFile().
Matching STRING

Match the STRING command like you would match HEADER. However, you must use two regular expressions, one to match strings in "double quotes", the other for matching strings in 'single quotes'.

if ($command -match '^STRING\s+"(.*)"$' -or $command -match "^STRING\s+'(.*)'$" ) { (1)
	$stringValue = $matches.1 (2)
}
1 Match STRING followed by a string in "double quotes" OR match STRING followed by a string in 'single quotes'.
2 Save the string to variable $stringValue
Matching WORD

Match the WORD command and extract the label and the filename with a regular expression. In this lab, the filename will use only double-quotes as delimiters.

if ($command -match '^WORD\s+(.*)\s+"(.*)"$') { (1)
	$WORDLabel = $matches.1 (2)
	$WORDFilename = $matches.2 (3)
}
1 Search the $command variable for “WORD” followed by whitespace followed by a string of non-blank characters (the label) followed by something in double-quote marks (the file name), using a regular expression.
2 Save the label to variable $WORDLabel
3 Save the filename to variable $WORDFilename
Matching INTEGER

Match the INTEGER command and extract the label, min and max with a regular expression.

if ($command -match '^INTEGER\s+(\w+)\s+(\d+)\s+(\d+)$') { (1)
	$integerLabel = $matches.1 (2)
	$integerMin = $($matches.2).toInt32($null) (3)
	$integerMax = $($matches.3).toInt32($null) (4)
}
1 Search the $command variable for “INTEGER” followed by whitespace followed by a string of word characters (label) followed by a string of digits (min) followed by a string of digits (max) using a regular expression.
2 Save the label to variable $integerLabel
3 convert the min string to an integer and save it to variable $integerMin
4 convert the max string to an integer and save it to variable $integerMin
Matching REFER

Match the REFER command and extract the label using a regular expression.

if ($command -match '^REFER\s+(\w+)$' ) { (1)
	$referLabel = $matches.1 (2)
}
1 Search the $command variable for “INTEGER” followed by whitespace followed by a string of word characters (label) using a regular expression.
2 Save the label to variable $referLabel
